TITLE:
🚀 Feature: APPWRITE_FUNCTION_EVENT_DATA should contain change

ISSUE BODY:
### 👟 Reproduction steps

perform change/update to your DB


### 👍 Expected behavior

triggered function should receive "a change"
ie OLD data vs NEW data

### 👎 Actual Behavior

You will get dump of updated data, but you have no idea what was there before.
So how we should "act" onUpdate?

To me, there is no need for another key. The data should be filled *only* with the data that has *actually* changed, and not the whole document.

ie: my document is {"name": "Seb", "username": "elsuperseby2"}

if the change happens on name I shoud receive as event data 

{"name": "Sebastien"} 

for instance.

One may argue for the need to know the previous values but I don't have any use case in mind :)
