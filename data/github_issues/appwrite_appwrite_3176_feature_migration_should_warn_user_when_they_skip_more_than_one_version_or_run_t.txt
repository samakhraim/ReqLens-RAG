TITLE:
🚀 Feature: Migration should warn user when they skip more than one version (or run twice).

ISSUE BODY:
### 🔖 Feature description

Migration script currently allows the user to break their deployment by upgrading too many major versions. 

For example, 0.11->0.12 is fine, 0.11->0.13 should not break the user's deployment.

The script should either not run, handle the situation gracefully, or error out when it's being used incorrectly.

TL;DR The migration script should not allow users to upgrade more than 1 major version at a time and break their deployments.

### 🎤 Pitch

As we release new versions more frequently and the product becomes more mature, it's important to safeguard users from accidentally upgrading more than one major version at a time and breaking their deployment. This is easily avoided by the migration script checking the current and target versions.
