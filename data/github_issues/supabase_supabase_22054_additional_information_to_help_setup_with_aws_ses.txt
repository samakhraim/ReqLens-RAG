TITLE:
Additional Information to help setup with AWS SES

ISSUE BODY:
# Improve documentation

## Link 

https://supabase.com/docs/guides/auth/auth-smtp

## Describe the problem

AWS SES has some additional requirements that can be confusing to those used to services like Resend or Sendgrid, such as:
- being placed in a testing sandbox initially and needing to request to leave
- requiring a verified domain

## Describe the improvement

We can add this information as a callout when using SES, with links to AWS documentation, to catch these issues early if people run into them

Add any other context or screenshots that help clarify your question.
