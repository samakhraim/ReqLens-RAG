PROJECT: Clinic Appointment System

The client wants a web and mobile system for a small medical clinic. Patients should be able to register, login, view doctors, see available appointment times, and book appointments.

Doctors should have profiles that include name, specialization, available days, available hours, consultation fee, and short biography.

The clinic admin should manage doctors, appointment schedules, patients, and appointment statuses.

The system should support Arabic and English. The client wants the interface to be simple because many patients are not technical.

Online payment is not required in the first version, but the client may add it later.

The client wants patients to receive appointment confirmation notifications. Notifications can be through email at first, and SMS may be added later.

The client is not sure if patients should be allowed to cancel appointments themselves or if cancellation should require admin approval.
